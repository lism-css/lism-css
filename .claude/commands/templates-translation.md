# Templates Translation Sync

`apps/docs/src/pages/preview/templates/` 配下のテンプレートプレビューを英語に翻訳し、同期します。

日本語（`index.astro`）を正として、英語（`en.astro`）を同期先とします。
また、`apps/docs/src/config/templates.ts` の description 英語版も同期対象とします。


## 対象範囲

- `$ARGUMENTS` でカテゴリが指定されている場合はそのカテゴリのみ対象とする
- 未指定の場合は全カテゴリを対象とする


## 作業手順

### 1. ファイル一覧の取得と差分分類

各テンプレートディレクトリ内の `index.astro` と `en.astro` を比較し、以下の3カテゴリに分類する：

| カテゴリ | 条件 | アクション |
|----------|------|------------|
| **新規** | `index.astro` があるが `en.astro` がない | テキスト部分を英訳して `en.astro` を作成 |
| **更新候補** | `index.astro` と `en.astro` の両方がある | コンポーネント構造を比較し、差異があれば更新 |
| **孤立** | `en.astro` があるが `index.astro` がない | 削除対象として報告 |

#### 更新候補の差分検出

`index.astro` と `en.astro` の両方を読み込み、以下の観点で差分を判定する：

- **構造的な差分**: コンポーネント・import の追加・削除・変更
- **Props の差分**: レスポンシブ値やスタイル props の変更
- **CSS の差分**: `_style.css` の変更（`en.astro` は共有なので構造反映のみ）

翻訳文の文言だけが異なる場合（日本語テキストが変わっていない場合）は更新不要。


### 2. 分類結果の報告

分類結果をユーザーに報告し、実行の承認を得る。

```
## テンプレート翻訳同期プラン

### 新規作成（{n}件）
- {category}/{id}/

### 更新（{n}件）
- {category}/{id}/ — {差分の概要}

### 削除（{n}件）
- {category}/{id}/en.astro（対応する index.astro なし）

### スキップ（{n}件）
- {category}/{id}/ — 差分なし

実行してよろしいですか？
```


### 3. 翻訳の実行

承認後、以下を実行する：

#### 3a. 孤立ファイルの削除

対応する `index.astro` がない `en.astro` を削除する。

#### 3b. 翻訳サブエージェントの起動

新規・更新対象のテンプレートを以下のグループに分けてサブエージェントを並列起動する：

| グループ | カテゴリ |
|----------|---------|
| A | `cta/`, `feature/`, `greeting/`, `history/` |
| B | `information/`, `member/`, `navigation/`, `news/` |
| C | `pricetable/`, `section/`, `testimonials/`, `works/` |

※ 対象ファイル数が少ない場合はグループを統合してよい

各サブエージェントに伝える情報：
- 担当テンプレートの一覧（カテゴリ/ID）
- 各テンプレートが「新規作成」か「更新」か
- 更新の場合は検出された差分の概要

#### 3c. templates.ts の description 同期

`apps/docs/src/config/templates.ts` の各テンプレートの `description.en` について：
- 新規テンプレートの場合は英語 description を追加
- 日本語 description が変更された場合は英語版も更新


### 4. 結果の報告

各サブエージェントの結果をまとめて報告する。

```
## テンプレート翻訳同期結果

### 新規作成（{n}件）
- {category}/{id}/en.astro

### 更新（{n}件）
- {category}/{id}/en.astro — {変更概要}

### 削除（{n}件）
- {category}/{id}/en.astro — 削除済み

### スキップ（{n}件）
```


## 翻訳ルール

### 基本方針
- コンポーネント構造・import・props は一切変更しない
- テキスト部分（見出し・本文・ボタンラベル等）のみ英訳する
- `_style.css` のインポートやCSSクラス名は変更しない
- `DemoLayout` の `title` prop はそのまま維持する
- URLはそのまま維持する
- alt="" のような空の alt はそのまま維持する

### テキストの翻訳方針
- 直訳ではなく、英語として自然な表現を使う
- ダミーテキストは英語の自然なダミーテキストに置き換える
- コード内のコメントは翻訳しない

### 日本語固有表現の英語化

**en.astro に日本語文字（漢字・ひらがな・カタカナ）が残ってはいけない。** 以下を必ず英語に変換すること：

- **人名**: 漢字の人名はローマ字（姓→名の順）に変換する（例: 佐藤 美咲 → Misaki Sato）
- **日付**: 「2020年1月1日」→ "January 1, 2020" のように英語の日付形式に変換する
- **肩書き・役職**: 「代表取締役社長」→ "CEO"、「部長」→ "Director" など英語の一般的な役職名に変換する
- **住所**: 日本語の住所は英語のダミー住所に置き換える
- **電話番号**: そのまま維持してよい

### 最終チェック

翻訳完了後、全ての `en.astro` ファイルに日本語文字（漢字・ひらがな・カタカナ）が残っていないことを以下のコマンドで確認する：

```bash
grep -rn '[一-龥ぁ-んァ-ヶー]' */*/en.astro | grep -v 'import\|from\|//'
```

出力がゼロ件であることを確認してから完了報告する。
