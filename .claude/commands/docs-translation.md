# Translation Sync

`apps/docs/src/content/ja/` の MDX ドキュメントを英語に翻訳し、`apps/docs/src/content/en/` に同期します。

日本語（`ja/`）を正として、英語（`en/`）を同期先とします。


## 対象範囲

- `$ARGUMENTS` でファイルやディレクトリが指定されている場合はその範囲のみ対象とする
- `root` が指定された場合は `ja/` 直下の mdx ファイルのみ対象とする（サブディレクトリは除外）
- 未指定の場合は `ja/` 配下の全 `.mdx` ファイルを対象とする
- ただし `_demo/` 配下は常に除外する


## 作業手順

### 1. ファイル一覧の取得と差分分類

`ja/` と `en/` の MDX ファイル一覧を取得し、以下の3カテゴリに分類する：

| カテゴリ | 条件 | アクション |
|----------|------|------------|
| **新規** | `ja/` にあるが `en/` にない | 翻訳して `en/` に作成 |
| **更新候補** | `ja/` と `en/` の両方にある | 内容を比較し、差異があれば更新 |
| **孤立** | `en/` にあるが `ja/` にない | 削除対象として報告 |

#### 更新候補の差分検出

`ja/` ファイルと `en/` ファイルの両方を読み込み、以下の観点で差分を判定する：

- **構造的な差分**: セクション（見出し）の追加・削除・並び替え
- **コンテンツの差分**: コード例の変更、Props テーブルの変更、import 文の変更
- **Frontmatter の差分**: `order`, `draft` 等のメタフィールドの変更

差分がないファイルはスキップする。
翻訳文の文言だけが異なる場合（日本語の文が変わっていない場合）は更新不要。


### 2. 分類結果の報告

分類結果をユーザーに報告し、実行の承認を得る。

```
## 翻訳同期プラン

### 新規作成（{n}件）
- ja/path/to/file.mdx

### 更新（{n}件）
- ja/path/to/file.mdx — {差分の概要}

### 削除（{n}件）
- en/path/to/file.mdx（対応する ja/ ファイルなし）

### スキップ（{n}件）
- ja/path/to/file.mdx — 差分なし

実行してよろしいですか？
```


### 3. 翻訳の実行

承認後、以下を実行する：

#### 3a. 孤立ファイルの削除

`en/` にのみ存在するファイルを削除する。

#### 3b. 翻訳サブエージェントの起動

新規・更新対象のファイルを以下のグループに分けて `lism-docs-translator` サブエージェントを並列起動する：

| グループ | ディレクトリ |
|----------|-------------|
| A | `core-components/` |
| B | `modules/` |
| C | `props/` |
| D | `ui/`（`examples/` 除く） |
| E | `ui/examples/` |
| F | その他（ルート直下の mdx） |

※ ファイル数が少ないグループ（3件以下）は隣接グループと統合してよい

各サブエージェントに伝える情報：
- 担当ファイルパスの一覧
- 各ファイルが「新規作成」か「更新」か
- 更新の場合は検出された差分の概要


### 4. 結果の報告

各サブエージェントの結果をまとめて報告する。

```
## 翻訳同期結果

### 新規作成（{n}件）
- en/path/to/file.mdx — {英語タイトル}

### 更新（{n}件）
- en/path/to/file.mdx — {変更概要}

### 削除（{n}件）
- en/path/to/file.mdx — 削除済み

### スキップ（{n}件）
```


## 翻訳ガイドライン

翻訳の詳細なルールは `translation` スキル（`.claude/skills/translation/SKILL.md`）を参照すること。

主要な注意事項：
- ファイル名（slug）は変更しない
- コンポーネント名・HTML タグ・CSS プロパティは翻訳しない
- コード例内のコメントは翻訳してよい
- 直訳ではなく、英語技術ドキュメントとして自然な表現を使う
- URL・内部リンクはそのまま維持する
