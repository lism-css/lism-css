この記事は、ブログ本文のスタイリングを確認するためのダミーコンテンツです。Markdown から生成される代表的な HTML 要素を、できるだけ網羅的に並べています。実装した CSS が、見出し・段落・リスト・引用・コード・テーブルなどに対して意図通り効いているかをまとめてチェックする用途を想定しています。

## 段落とインライン要素

通常の段落テキストです。文中には **太字（strong）**、*斜体（em）*、`インラインコード（code）`、[リンク（a）](https://lism-css.com/)、~~取り消し線（del / s）~~ などの装飾を混ぜることができます。日本語と English text が混在したときの行間や字間も、ここで合わせて確認できます。

短い段落も用意しておきます。

連続した段落の上下マージンが破綻していないか、`Flow` 等のレイアウトで正しく流れているかを目視で確認するための行です。

### 強調の組み合わせ

**強調の中に `code` が混ざる** ケースや、*斜体の中の [リンク](https://example.com/)*、**[太字のリンク](https://example.com/)** など、装飾の入れ子も含めて確認しておきます。

## リスト

### 順不同リスト（ul）

- 一つ目の項目
- 二つ目の項目
- ネストを含む項目
  - 子項目 A
  - 子項目 B
    - 孫項目
- 最後の項目

### 順序付きリスト（ol）

1. 手順その一
2. 手順その二
3. ネストを含む手順
   1. サブステップ A
   2. サブステップ B
4. 手順その四

### タスクリスト

- [x] 完了済みのタスク
- [ ] 未完了のタスク
- [ ] もう一つの未完了タスク

## 引用（blockquote）

> 短い一文の引用。本文と区別され、左にボーダーが引かれることを期待しています。

> 複数行にわたる引用も確認しておきます。
>
> 段落をまたぐ場合に、引用全体としての余白がきちんと取られているか、ブロック内の段落間が詰まりすぎていないかをチェックします。
>
> *— 引用元の名前*

## コード

インラインの `const value = 42;` と、ブロックのコードを並べてみます。

```ts
// TypeScript のサンプル
type Post = {
  title: string;
  tags: string[];
  publishedAt: Date;
};

export function formatTags(post: Post): string {
  return post.tags.map((t) => `#${t}`).join(' ');
}
```

```css
/* CSS のサンプル */
.c--articleBody {
  font-size: var(--fz--s);
  line-height: 1.95;

  & h2 {
    font-size: var(--fz--xl);
    border-block-end: 1px solid var(--divider);
  }
}
```

## テーブル

| 要素 | 用途 | 備考 |
| --- | --- | --- |
| `h2` | 大見出し | 下線付き |
| `h3` | 中見出し | 左にアクセントボーダー |
| `p` | 段落 | 両端揃え |
| `blockquote` | 引用 | 左ボーダー＋斜体 |
| `pre` / `code` | コード | 等幅フォント |

## 区切り線

セクション間の区切りに使う `<hr>` の見え方も確認します。
