Utility classes defined in the `lism-base` layer follow the `set--{className}` format. They handle base styling for HTML elements and CSS variable setup.

The set-- classes fall into two categories:

| Category | Purpose | Classes |
|----------|---------|---------|
| Base styling | Resetting or reverting HTML element styles | `set--plain` / `set--revert` |
| Variable setup | Setting up CSS variables only | `set--bxsh` / `set--hov` / `set--bdrsInner` / `set--bleed` / `set--s` |

The `{name}` in `set--{name}` corresponds to the name of the CSS variable each class sets up (e.g. `--bxsh--*` → `bxsh`, `--_isHov` / `--_notHov` → `hov`, `--bdrs--inner` → `bdrsInner`, `--s*` → `s`).

## `set--revert`

A class for explicitly reverting to the browser's default appearance.
On its own it has no effect (it does not do a wholesale revert of every property).

It only takes effect on specific HTML elements, and currently only `ul` / `ol` are supported.

In Lism CSS, list styles are reset on `ul` / `ol` elements that have any class.
Apply `set--revert` when a list has classes but you still want to keep the list markers (for example, a list that only carries Property Classes).

:::info
Classless `ul` / `ol` elements (lists with no `class` attribute at all) automatically retain the browser's default list style, so `set--revert` is not needed.
:::


## `set--bxsh`

Elements with the `set--bxsh` class have the `--bxsh--` variables that compose the shadow redefined.

```css
:root,
.set--bxsh {
  --bxsh--10: var(--shsz--10) var(--shc);
  --bxsh--20: var(--shsz--20) var(--shc);
  --bxsh--30: var(--shsz--30) var(--shc);
  --bxsh--40: var(--shsz--40) var(--shc);
  --bxsh--50: var(--shsz--50) var(--shc);
}
```

By overriding the `--shc` variable here, you can adjust the color tint of the shadow.

## `set--s`

A class that recalculates spacing tokens (`--s5` through `--s80`) against the current font size. Use it for parts like Buttons or Badges where internal spacing should follow the element's own font-size.

```css
:root,
.set--s {
  --s5: calc(var(--s-unit) * 0.5);
  --s10: var(--s-unit);
  --s15: calc(var(--s-unit) * 1.5);
  --s20: calc(var(--s-unit) * 2);
  --s25: calc(var(--s-unit) * 2.5);
  --s30: calc(var(--s-unit) * 3);
  --s35: calc(var(--s-unit) * 4);
  --s40: calc(var(--s-unit) * 5);
  --s50: calc(var(--s-unit) * 8);
  --s60: calc(var(--s-unit) * 13);
  --s70: calc(var(--s-unit) * 21);
  --s80: calc(var(--s-unit) * 34);
}
```

```html
<button class="set--s -fz:s -py:10 -px:20">...</button>
```

For details, see [Spacing Tokens](/en/docs/tokens/spacing/).

## `set--hov`

Sets the hover state variables (`--_notHov`, `--_isHov`). Primarily used to **control hover styles on child elements**.

- `--_isHov` is defined as an empty value only when not hovered (and not focus-within).
- Conversely, `--_notHov` is defined as an empty value when hovered or focus-within.

<SrcCode path="src/scss/base/set/_hov.scss" />


This empty variable trick enables the following patterns:

- `var(--_isHov, {value to apply on hover})`
- `var(--_notHov, {value to apply when not hovered})`

For example, with `color: var(--_isHov, red)`, when hovered `--_isHov` becomes undefined, so `color: red` is applied.

:::info
Using `var(--_isHov, ...) var(--_notHov, ...)` allows you to **write both hover and non-hover styles on a single line**, but this does not work for properties like `box-shadow` that accept space-separated values.
:::

<PreviewTitle>Example: `-hov:test`</PreviewTitle>
```css
.-hov\:test {
  color: var(--_isHov, green) var(--_notHov, red); /* green on hover, red when not hovered */
  box-shadow: var(--_isHov, var(--bxsh--30)); /* shadow on hover (invalid value when not hovered) */
  opacity: var(--_notHov, var(--o--pp)); /* reduced opacity when not hovered (invalid value on hover) */
}
```

By applying `set--hov` to a parent element and using the `--_isHov` and `--_notHov` variables on child elements, you can trigger style changes on children based on the parent's hover state.

Use this, for example, to zoom an image inside a card when the parent box is hovered. The `-hov:in:zoom` class is provided out of the box for this purpose.

```css
.-hov\:in\:zoom {
  --transitionProps: scale;
  scale: var(--_isHov, 1.1);
}
```

## `set--bleed`

A class that generates the `--bleed` variable, useful for building one-side bleed layouts.
The amount of space outside the content width (`--contentSize`) is set as its value. The gutter width (`--gutter` / `--gutter--base`) acts as the lower bound — if the computed value falls below the gutter width, the gutter width is returned instead.

## `set--bdrsInner`

A class that automatically calculates the inner border radius (`--bdrs--inner`) from the parent element's border radius (`--bdrs`) and padding (`--p`).
Apply `set--bdrsInner` to the parent element, then use `-bdrs:inner` on the child element to apply the calculated `--bdrs--inner`.

(Both `bdrs` and `p` must be specified on the parent element.)
