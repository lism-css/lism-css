---
title: '-max-sz'
description: 'How to use the -max-sz class for controlling the maximum width of content.'
---


Lism provides content size tokens in the form `--sz--{sz}` (where `{sz}` = `xs`, `s`, `m`, `l`, `xl`), and corresponding `-max-sz` classes are available for each. (`-max-sz:{xs~xl}`)

| Class | Value |
|---|---|
| `-max-sz:xs` | `var(--sz--xs)` |
| `-max-sz:s` | `var(--sz--s)` |
| `-max-sz:m` | `var(--sz--m)` |
| `-max-sz:l` | `var(--sz--l)` |
| `-max-sz:xl` | `var(--sz--xl)` |


## Expanding to fill the parent or container

The behavior previously provided as `-max-sz:full` / `-max-sz:container` has been reorganized as Trait Classes.

| Purpose | Class |
|---|---|
| Fill the parent element (including past `has--gutter`) | <ModLink slug="is--fullwide" /> |
| Expand up to the nearest `is--container` width (centered) | <ModLink slug="is--bleed" /> |
