import { Lism, Spacer } from "lism-css/astro";

A general-purpose component that accepts a wide variety of props aligned with the Lism CSS design system.
When no props are specified, it simply renders a `<div>`.

:::info
Nearly all components distributed by Lism are based on this `<Lism>` component.
:::

## Lism Props

The props specific to Lism CSS that `<Lism>` accepts are called **Lism Props**.

{/* In addition to the above, there are several more complex ones. These are explained in the [Notes](#notes) section below. */}


{/* Lism Props are mainly classified into the following three types: */}

{/* |Type|Description|
|---|---|
|**Common Props**|Basic shared properties|
|**Trait Props**|Props for outputting classes classified as [Trait Primitives](/en/docs/css-methodology//#trait-primitives)|
|**CSS Props**|Props for outputting classes classified as [Property Class](/en/docs/property-class/)| */}



### `lismClass`

Specifies the primary class name for the current component.
This class is output with a relatively high priority in the `<Lism>` class processing order.

<PreviewTitle>Example</PreviewTitle>
```jsx 'lismClass="c--myComponent"'
<Lism p="10" lismClass="c--myComponent">Lorem ipsum texts...</Lism>

↓

<div class="c--myComponent -p:10">Lorem ipsum texts...</div>
```


### `variant`

Outputs a variation class for `lismClass`.
When multiple classes are specified in `lismClass`, the variation class is applied to the first class.


<PreviewTitle>Example</PreviewTitle>
```jsx 'variant="secondary"'
<Lism lismClass="c--myComponent" variant="secondary">Lorem ipsum texts...</Lism>

↓

<div class="c--myComponent c--myComponent--secondary">Lorem ipsum texts...</div>
```


### layout

Specifies a layout primitive. For example, setting `layout='flow'` outputs the `.l--flow` class.

<PreviewTitle>Example</PreviewTitle>
```jsx 'layout="flow"'
<Lism layout="flow">Lorem ipsum texts...</Lism>

↓

<div class="l--flow">Lorem ipsum texts...</div>
```


### `set`

Specifies a [Set Class](/en/docs/set-class/). The given values are output as `set--{value}` classes.

<PreviewTitle>Example</PreviewTitle>
```jsx 'set="hov"'
<Lism set="hov">...</Lism>

↓

<div class="set--hov">...</div>
```

#### Multiple values

Multiple values can be specified as a space-separated string.

```jsx
<Lism set="hov transition">...</Lism>
```

#### Excluding with `-` prefix

Prefix a value with `-` to exclude that identifier. This is useful when you want to remove a set class that has been applied internally by a component.

```jsx
{/* Remove the set--plain that AccordionButton applies internally */}
<AccordionButton set="-plain">...</AccordionButton>

{/* Enable hov and transition while excluding shadow */}
<Lism set="hov transition shadow -shadow">...</Lism>
```


### `util`

Specifies `u--` utility classes. The given values are output as `u--{value}` classes.

<PreviewTitle>Example</PreviewTitle>
```jsx 'util="cbox"'
<Lism util="cbox">...</Lism>

↓

<div class="u--cbox">...</div>
```

Like the `set` prop, multiple values can be specified as a space-separated string, and a `-` prefix excludes that identifier.

```jsx
<Lism util="cbox trim">...</Lism>
<Lism util="cbox -trim">...</Lism>
```


### `as`

Specifies the HTML tag or external component to render. Defaults to `div`.

<PreviewTitle>Example of specifying an HTML tag</PreviewTitle>
```jsx 'as="p"'
<Lism as="p">Lorem ipsum texts...</Lism>

↓

<p>Lorem ipsum texts...</p>
```

By specifying an external component, you can have `<Lism>` **render as a different external component**.

<Preview>
  <PreviewTitle>Example: rendering Lism's `<Media>` component using Next.js's `<Image>`</PreviewTitle>
  <PreviewCode>
```jsx
import Image from 'next/image';

<Media as={Image} src="..." p="20" bd />
```
  </PreviewCode>
</Preview>

Lism Props (`p` and `bd` in the example above) are first processed to generate `className` and `style`, which are then passed to the component specified by `as`.
This is useful when you want to use Lism Props on non-Lism components.


### `exProps`

Accepts **props that you want to skip Lism Props processing for** as an object.

This is handy when a prop name of the external component specified via `as` conflicts with a Lism Prop name, and you want to explicitly mark it as belonging to the external component.

<Preview>
  <PreviewTitle>Example</PreviewTitle>
  <PreviewCode>
```jsx
<Icon as={Hoge} exProps={{ size: '1em' }} p="10" fz="l">
  ...
</Icon>
```
  </PreviewCode>
</Preview>

By doing this, the Lism `Icon` component processes `p` and `fz`, while the external component `Hoge` reliably receives `size`.
(In this particular example, `exProps` is not strictly necessary since Lism does not currently process `size`, but explicitly marking external component props ensures no future conflicts.)



### Trait Props

A set of props corresponding to [Trait Primitives](/en/docs/css-methodology/) classes.


| Prop | Output class | Purpose |
|---|---|---|
| `isWrapper(='{s\|l}')` | `is--wrapper` + `-contentSize:{s\|l}` | Content width restriction |
| `isLayer` | `is--layer` | Absolute positioned layer (inset:0) |
| `isBoxLink` | `is--boxLink` | Make entire box clickable |
| `isCoverLink` | `is--coverLink` | Link with a clickable area covering the parent element |
| `isContainer` | `is--container` | Container query target |
| `isSide` | `is--side` | Side element |
| `isSkipFlow` | `is--skipFlow` | Skip Flow spacing |
| `isVertical` | `is--vertical` | Vertical writing mode |

<Reference>
[Learn about Trait Primitives classes and usage examples](/en/docs/css-methodology/)
</Reference>


### CSS Props

Major CSS properties can be specified using shorthand notation.
For example, `font-size` can be written as `fz`, and `padding` as `p`.

<Reference>
[See the full list of available Props](/en/docs/property-class/)
</Reference>

<Spacer h="10"/>

The output format is determined by the value (`value`) specified via `prop={value}`:

| Value | Output format | Example |
|------|------|-----|
| Token / preset value | `-{prop}:{value}` class | `fz='l'` → `-fz:l` |
| `true` / `"-"` | `-{prop}` class only (no variable) | `bd` → `-bd` |
| Value starting with `:` | Forced utility class | `p=':hoge'` → `-p:hoge` |
| Other values (BP-supported) | `-{prop}` class + `--{prop}` variable | `fz='20px'` → `-fz` + `--fz:20px` |
| Other values (BP-unsupported) | Direct `style` attribute output | `o='0.7'` → `opacity:0.7` |
| Other values (variable-only) | `--{prop}` variable only | `bdw='2px'` → `--bdw:2px` |

:::note[Notes]
- **BP-supported properties**: Properties that can switch values at breakpoints. See the **BP** column in [Property Class](/en/docs/property-class).
- **Variable-only properties**: Properties like `bds`, `bdc`, `bdw`, `keycolor` that always output only CSS variables (`--{prop}`).
- When a matching token exists but has no dedicated class, the value is output as a CSS variable. (e.g., `c='red'` → `class="-c"` + `style="--c:var(--red)"`)
:::

#### Force output as a utility class with `:`

Even if no corresponding Property Class exists, prefixing a value with `:` will output the string after it as a utility class name.

```jsx
<Lism p=":hoge">...</Lism>
// → <div class="-p:hoge">...</div>
```

Add a custom utility class in your CSS and use it as needed.

```css
.-p\:hoge {
  /* ... your styles ... */
}
```

:::info
Using `className='-p:hoge'` also works, but passing it as a CSS Prop via `:` ensures consistent output ordering.
:::


#### Responsive syntax

BP-supported properties can specify values per breakpoint (`sm`, `md`) using arrays or objects.

[Learn more about responsive support](/en/docs/responsive)

<Preview>
  <PreviewTitle>Example 1: array syntax (base → sm → md)</PreviewTitle>
  <PreviewCode label="JSX" slot="tab">
    ```jsx
    <Lism p={['20', '30', '5rem']}>...</Lism>
    ```
  </PreviewCode>
  <PreviewCode label="Output HTML" slot="tab">
    ```html
    <div class="-p:20 -p_sm -p_md" style="--p_sm:var(--s30);--p_md:5rem">
      ...
    </div>
    ```
  </PreviewCode>
</Preview>

<Preview>
  <PreviewTitle>Example 2: skip intermediate breakpoints (`@md` only)</PreviewTitle>
  <PreviewCode label="JSX" slot="tab">
    ```jsx
    <Lism p={[null, null, '40']}>...</Lism>
    ```
  </PreviewCode>
  <PreviewCode label="Output HTML" slot="tab">
    ```html
    <div class="-p_md" style="--p_md:var(--s40)">
      ...
    </div>
    ```
  </PreviewCode>
</Preview>

:::warning
Whether a value actually changes at a breakpoint depends on whether that property supports responsive behavior.
Check the **BP** (<small>short for breakpoint</small>) column in [Property Class](/en/docs/property-class) to see which properties are supported.
:::

For properties that do not support responsive behavior by default (i.e., no CSS is provided), you can add support via [SCSS customization](/en/docs/customize/).


### Class output order

The `class` attribute generated by `<Lism>` is broadly divided into three blocks: **"user-specified", "components & primitives", and "utilities"**. The utility block is ordered from coarser to finer granularity: `set--` → `u--` → Property Class.

```
[className] [lismClass] [l--{layout}] [is--*(Trait)] [set--*] [u--*] [Property Class...]
```

| # | Category | Source | Example |
|---|---|---|---|
| 1 | `className` | Classes passed directly by the user | `my-card` |
| 2 | `lismClass` (+ `variant`) | The component's own class, such as `c--*` (composite) or `a--*` (atomic) | `c--box`, `a--icon`, `c--box c--box--primary` |
| 3 | Layout Primitive | `layout` prop | `l--box`, `l--flow` |
| 4 | Trait Primitive | Trait Props (`isWrapper`, etc.) | `is--wrapper`, `is--layer` |
| 5 | Set Class | `set` prop | `set--hov`, `set--card` |
| 6 | `u--` utilities | `util` prop | `u--cbox`, `u--trim` |
| 7 | Property Class | CSS Props (including `hov`) | `-p:20`, `-bgc:base-2` |

Items 2–4 all belong to [Component Class / Primitive Class](/en/docs/css-methodology/) (CSS layer `lism-component` / `lism-primitive`), while 5–7 are utility-type classes.

:::note
The `-flow:{size}` class produced by `layout="flow"` is treated as a layout-primitive-specific configuration class, so it is emitted directly after `l--flow`.
:::

:::note
The order within the `class` attribute does not affect the CSS cascade (specificity or cascade order). This ordering exists purely for readability and consistency.
:::
