`<Media>` is a Lism component for outputting media elements such as images and videos.  
It outputs an `<img>` tag by default.

## Props

| Property | Description |
|---|---|
| `objectPosition` | Specifies the `object-position` of the media element. |
| `objectFit` | Specifies the `object-fit` of the media element. |
| `blur`,`contrast`,`brightness`,`dropShadow`,`grayscale`,`hueRotate`,`invert`,`saturate`,`sepia` | These are concatenated together and passed to `style.filter`. |

#### Using Next.js `<Image>`

Next.js `Image` and similar components can be passed via the `as` property.

<PreviewTitle>Passing Next.js `Image` via `as`</PreviewTitle>
```jsx "as={Image}"
import { Media } from 'lism-css/react';
import Image from 'next/image';

<Media as={Image} src="/img.jpg" alt="" width="960" height="640" alt="Dummy Image" />;
```

#### Using Astro `<Image>`

Astro's `<Image>` cannot be passed directly via `as`.
An Astro (TypeScript?) check throws an "Image missing required 'alt' property." error.

Therefore, in `<Media>` imported from `lism-css/astro`, passing the string `'AstroImage'` to `as` will internally invoke `<Image>`.


<PreviewTitle>Example of using Astro's `<Image>`</PreviewTitle>
```jsx "as='AstroImage'"
import { Media } from 'lism-css/astro';

<Media as="AstroImage" src="/img.jpg" alt="Dummy Image" width="960" height="640" />;
```
