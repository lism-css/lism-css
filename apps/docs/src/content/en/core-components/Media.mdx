`<Media>` is a Lism component for outputting media elements such as images and videos.  
In the Astro version, Astro's `<Image>` is used by default. In the React version, it outputs an `<img>` tag by default.

#### Using Next.js `<Image>`

Next.js `Image` and similar components can be passed via the `as` property.

<PreviewTitle>Passing Next.js `Image` via `as`</PreviewTitle>
```jsx "as={Image}"
import { Media } from 'lism-css/react';
import Image from 'next/image';

<Media as={Image} src="/img.jpg" alt="Dummy Image" width="960" height="640" />
```

#### Using Astro `<Image>` / `<Picture>`

In `<Media>` imported from `lism-css/astro`, the component is automatically mapped based on the `as` value.

| `as` | Output |
|------|--------|
| omitted / `"img"` | Astro `<Image />` |
| `"picture"` | Astro `<Picture />` |
| `"video"` / `"iframe"` etc. | HTML tag as-is |

<PreviewTitle>Example of using Astro's `<Image>` (default)</PreviewTitle>
```jsx
import { Media } from 'lism-css/astro';

<Media src="/img.jpg" alt="Dummy Image" width="960" height="640" />
```

<PreviewTitle>Example of using Astro's `<Picture>`</PreviewTitle>
```jsx "as='picture'"
import { Media } from 'lism-css/astro';

<Media as="picture" src="/img.jpg" alt="Dummy Image" width="960" height="640" />
```
