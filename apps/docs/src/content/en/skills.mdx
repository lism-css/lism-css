By placing the SKILL.md file in your project, you can communicate Lism CSS design rules, Props, tokens, and naming conventions to AI agents.


GitHub: [lism-css/skills/lism-css-guide/](https://github.com/lism-css/lism-css/tree/main/skills/lism-css-guide)



## Setup

<Preview>
  <PreviewCode slot="tab" label="npm">
  ```bash frame="none"
  npx skills add lism-css/lism-css
  ```
  </PreviewCode>
  <PreviewCode slot="tab" label="pnpm">

Install the Lism CSS skill into your project using the command above.


<small>For more information about the `skills` command, see [skills.sh](https://skills.sh).</small>


### Update

If installed globally, you can update with the following commands:

```bash frame="none"
# Check for available updates
npx skills check

# Update skills to the latest version
npx skills update
```

If installed in a project, run `skills add` again. To check for updates, see the [GitHub repository](https://github.com/lism-css/lism-css/tree/main/skills/lism-css-guide) directly.

```bash frame="none"
npx skills add lism-css/lism-css
```

### File structure

The skill consists of 10 files:

| File | Contents |
|------|----------|
| `SKILL.md` | Entry point — overview, package structure, implementation rules, reference links |
| `tokens.md` | Design tokens — spacing, font sizes, border radius, shadows, colors, palettes |
| `base-styles.md` | Base styling — Reset CSS, HTML element base styles, CSS variables overview |
| `set-class.md` | `set--` classes — `set--plain`/`set--shadow`/`set--hov`/`set--transition`, etc. |
| `primitive-class.md` | Primitive classes — is--/l--/a-- and Component (c--) class list and usage |
| `utility-class.md` | Utility classes — u-- class list |
| `property-class.md` | Property Class — `-{prop}:{value}` syntax, key Prop list |
| `prop-responsive.md` | Responsive design — breakpoints, container queries, usage patterns |
| `components.md` | Component system — core, layout, semantic, and UI component list |
| `css-rules.md` | CSS design rules — Layer structure, naming conventions, prefixes, custom CSS rules |
