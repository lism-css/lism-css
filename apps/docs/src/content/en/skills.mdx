By placing the SKILL.md file in your project, you can communicate Lism CSS design rules, Props, tokens, and naming conventions to AI agents.


GitHub: [lism-css/skills/lism-css-guide/](https://github.com/lism-css/lism-css/tree/main/skills/lism-css-guide)


## Setup

`@lism-css/cli`'s `lism skill add` can auto-detect the AI tools you use and place the SKILL.md files accordingly.

In interactive mode, the CLI detects project-root markers (`.claude/`, `.cursor/`, etc.) and pre-checks matching destinations. To target tools non-interactively, pass the flags directly:

```bash frame="none"
# Single tool
npx @lism-css/cli skill add --claude

# Multiple tools
npx @lism-css/cli skill add --claude --cursor

# All tools
npx @lism-css/cli skill add --all
```

### Destinations

| Flag | Destination |
|------|-------------|
| `--claude` | `.claude/skills/lism-css-guide` |
| `--codex` | `.agents/skills/lism-css-guide` |
| `--cursor` | `.cursor/skills/lism-css-guide` |
| `--windsurf` | `.windsurf/skills/lism-css-guide` |
| `--cline` | `.cline/skills/lism-css-guide` |
| `--copilot` | `.github/skills/lism-css-guide` |
| `--gemini` | `.gemini/skills/lism-css-guide` |
| `--junie` | `.junie/skills/lism-css-guide` |


### Update

You can check for version drift and force-update as needed.

```bash frame="none"
# Compare bundled version vs local version
npx @lism-css/cli skill check

# Overwrite with the latest version
npx @lism-css/cli skill update --claude
```

For upstream changes, see the [GitHub repository](https://github.com/lism-css/lism-css/tree/main/skills/lism-css-guide) directly.


### Using [skills.sh](https://skills.sh)

If you are using [skills.sh](https://skills.sh), `skills add lism-css/lism-css` also fetches the same skill.

```bash frame="none"
npx skills add lism-css/lism-css
```


### File structure

The skill consists of 10 files:

| File | Contents |
|------|----------|
| `SKILL.md` | Entry point — overview, package structure, implementation rules, reference links |
| `tokens.md` | Design tokens — spacing, font sizes, border radius, shadows, colors, palettes |
| `base-styles.md` | Base styling — Reset CSS, HTML element base styles, CSS variables overview |
| `set-class.md` | `set--` classes — `set--plain`/`set--shadow`/`set--hov`/`set--transition`, etc. |
| `primitive-class.md` | Primitive classes — is--/l--/a-- and Component (c--) class list and usage |
| `utility-class.md` | Utility classes — u-- class list |
| `property-class.md` | Property Class — `-{prop}:{value}` syntax, key Prop list |
| `prop-responsive.md` | Responsive design — breakpoints, container queries, usage patterns |
| `components.md` | Component system — core, layout, semantic, and UI component list |
| `css-rules.md` | CSS design rules — Layer structure, naming conventions, prefixes, custom CSS rules |
