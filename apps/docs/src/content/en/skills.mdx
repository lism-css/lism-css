By placing the SKILL.md file in your project, you can communicate Lism CSS design rules, Props, tokens, and naming conventions to AI agents.


GitHub: [lism-css/skills/lism-css-guide/](https://github.com/lism-css/lism-css/tree/main/skills/lism-css-guide)


## Setup

`@lism-css/cli`'s `lism skill add` can auto-detect the AI tools you use and place the SKILL.md files accordingly.

In interactive mode, the CLI detects project-root markers (`.claude/`, `.cursor/`, etc.) and pre-checks matching destinations. To target tools non-interactively, pass the flags directly:

```bash frame="none"
# Single tool
npx @lism-css/cli skill add --claude

# Multiple tools
npx @lism-css/cli skill add --claude --cursor

# All tools
npx @lism-css/cli skill add --all
```

### Update

You can diff your local files against the bundled skill and force-update as needed.

```bash frame="none"
# Diff the bundled skill against local files (shows changed / added / removed summary)
npx @lism-css/cli skill check

# Overwrite with the latest version
npx @lism-css/cli skill update --claude
```

`skill check` compares each installed file with the bundled one by sha256, then lists the changed / added / removed files. For upstream changes, see the [GitHub repository](https://github.com/lism-css/lism-css/tree/main/skills/lism-css-guide) directly.


### Using [skills.sh](https://skills.sh)

If you are using [skills.sh](https://skills.sh), `skills add lism-css/lism-css` also fetches the same skill.

```bash frame="none"
npx skills add lism-css/lism-css
```


### File structure

The skill consists of 13 files:

| File | Contents |
|------|----------|
| `SKILL.md` | Entry point ŌĆö overview, package structure, implementation rules, reference links |
| `tokens.md` | Design tokens ŌĆö spacing, font sizes, border radius, shadows, colors, palettes |
| `base-styles.md` | Base styling ŌĆö Reset CSS, HTML element base styles, CSS variables overview |
| `set-class.md` | `set--` classes ŌĆö `set--plain`/`set--shadow`/`set--hov`/`set--transition`, etc. |
| `primitive-class.md` | Primitive classes ŌĆö is--/l--/a-- and Component (c--) class list and usage |
| `utility-class.md` | Utility classes ŌĆö u-- class list |
| `property-class.md` | Property Class ŌĆö `-{prop}:{value}` syntax, key Prop list |
| `prop-responsive.md` | Responsive design ŌĆö breakpoints, container queries, usage patterns |
| `components-core.md` | `lism-css` components ŌĆö Lism / layout / semantic / Atomic & Trait Primitives |
| `components-ui.md` | `@lism-css/ui` components ŌĆö Accordion / Modal / Tabs / Button / Badge / Callout, etc. |
| `css-rules.md` | CSS design rules ŌĆö Layer structure, prefixes, custom CSS rules |
| `naming.md` | Naming conventions ŌĆö CSS variables / classes / `{prop}` / `{value}` shortening rules |
| `customize.md` | Customization ŌĆö toggling `@layer`, SCSS / `lism.config.js` overrides, extra styles |
