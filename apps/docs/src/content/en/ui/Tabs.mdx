A component for creating tab elements.
It provides minimal styling — only the interaction behavior is included. Visual styles should be customized per project.


## How to use

Provided as `Tabs` from the `@lism-css/ui` package.


### Import

<ImportPackage component="Tabs" script="tabs.js" packageName="@lism-css/ui" />


### Props

| Property | Description |
|---|---|
|<PropBadge>`<Tabs.Root>`</PropBadge><br/>`variant`| Outputs the `c--tabs--{variant}` class. |
|<PropBadge>`<Tabs.Root>`</PropBadge><br/>`tabId`| A string ID used as the value for `aria-controls` to identify each tab. |
|<PropBadge>`<Tabs.Root>`</PropBadge><br/>`defaultIndex`| Specifies which tab to open initially. |
|<PropBadge>`<Tabs.Root>`</PropBadge><br/>`listProps`| Props passed to the list element (`c--tabs_list`) wrapping the tab buttons. |


### Structure

```jsx
<Tabs.Root>
  <Tabs.Item>
    <Tabs.Tab>Tab label</Tabs.Tab>
    <Tabs.Panel>...Contents...</Tabs.Panel>
  </Tabs.Item>
</Tabs.Root>
```

:::warning
Only `<Tabs.Tab>` and `<Tabs.Panel>` are recognized as children of `<Tabs.Item>`. Both must always be used together.
:::


### Source Code

The source code is available on [GitHub](https://github.com/lism-css/lism-css/tree/main/packages/lism-ui/src/components/Tabs).

### Displaying tab content side by side

Setting `grid-template: 'list panel' auto / auto 1fr;` on `<Tabs.Root>` (`.c--tabs`) places the tab list and content side by side.

At the same time, changing the tab list (`.c--tabs_list`) buttons to stack vertically (`flex-direction: column;`) achieves the following layout.

<Preview>
  <PreviewTitle>Example: side-by-side layout on `@sm` and above</PreviewTitle>
  <PreviewArea p="30" resize>
    <Tabs.Root gt={[null, '"list panel" auto / auto 1fr']} listProps={{ fxd: [null, 'column'] }}>
      <Tabs.Item>
        <Tabs.Tab>
          <Fragment>Tab 1</Fragment>
        </Tabs.Tab>
        <Tabs.Panel>
          <DummyText pre="Tab 01: " />
        </Tabs.Panel>
