## Examples of creating custom utility classes

While no classes are provided in the Lism package itself, here are examples of defining your own utility classes and using them.

This leverages the [force output as utility class](/components/lism#output-utility) approach.


<Preview>
  <PreviewTitle>stripe</PreviewTitle>
  <PreviewArea resize nogrid p='20'>
    <Box bg=":stripe" p="30">
      <Dummy />
    </Box>
    <Box bg=":stripe" p="30" className="u--cbox" keycolor="blue">
      <Dummy />
    </Box>
  </PreviewArea>
  <PreviewCode label='CSS' slot='tab'>
```css
.-bg\:stripe {
  --stripe-bdc: color-mix(in srgb, currentColor, transparent 94%);
  --stripe-bdw: 1px;
  --stripe-deg: -40deg;
  --stripe-interval: 6px;
  background-image: repeating-linear-gradient(
    var(--stripe-deg),
    var(--stripe-bdc),
    var(--stripe-bdc) var(--stripe-bdw),
    transparent var(--stripe-bdw),
    transparent var(--stripe-interval)
  );

  background-position: center;
}
```
  </PreviewCode>
    <PreviewCode label='JSX' slot='tab'>
    ```jsx "bg=':stripe'"
    <Box bg=':stripe' p='30'>
      <Dummy />
    </Box>
    <Box bg=':stripe' p='30' className='u--cbox' keycolor='blue'>
      <Dummy />
    </Box>
    ```
  </PreviewCode>
  <PreviewCode label='HTML' slot='tab'>
    ```html "-bg:stripe"
    <div class="l--box -bg:stripe -p:30">
      <p>...</p>
    </div>
    <div class="l--box u--cbox -bg:stripe -p:30 -keycolor:blue">
      <p>...</p>
    </div>
    ```
  </PreviewCode>
</Preview>
