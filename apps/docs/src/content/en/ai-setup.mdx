This guide explains how to set up AI coding tools (Claude Code, Cursor, Codex, etc.) to work effectively with Lism CSS.

There are two main approaches:

| Method | Description |
|--------|-------------|
| **MCP Server** | Allows AI tools to reference component, Props, and token information in real time |
| **Rule files** | Place a file in your project to communicate design rules to AI agents |

Either approach works on its own, but **using both together is more effective**.


## MCP Server

`@lism-css/mcp` is an MCP (Model Context Protocol) server that provides Lism CSS documentation and API information to AI coding tools.

### What it can do

- **Component reference** â€” Props, usage examples, and sub-component structure for each component
- **Props to CSS mapping** â€” Reverse lookup of abbreviations like `p` â†’ `padding`, `fz` â†’ `font-size`
- **Design token reference** â€” Token values for colors, spacing, font sizes, and more
- **Documentation search** â€” Full-text search by keywords
- **CSS to Lism conversion** â€” Suggests Lism CSS props and components from existing CSS code

## Rule files

By placing a rule file in your project, you can communicate Lism CSS design rules and usage patterns to AI agents.

Each tool uses a different file name:

| Tool | File | How to use |
|------|------|------------|
| Claude Code | `.claude/skills/lism-css/SKILL.md` | Auto-loaded as a skill |
| OpenAI Codex | `AGENTS.md` | Place in project root |
| Cursor | `.cursor/rules/lism-css.mdc` | Place as a rule file |
| GitHub Copilot | `.github/copilot-instructions.md` | Place in project root |

Copy the template below and place it as a rule file in your project.

:::note[For Claude Code]
When placed as `.claude/skills/lism-css/SKILL.md`, it is automatically recognized as a skill.
For other tools (Codex, Cursor, etc.), copy only the body content without the frontmatter section (the part enclosed in `---`).
:::

# Lism CSS Guide

This project uses the CSS framework "Lism CSS".

If the MCP server (`@lism-css/mcp`) is available, use it to get detailed component and Props information.

Official documentation: https://lism-css.com/en/docs/overview/


## Package structure

| Package | Purpose |
|---------|---------|
| `lism-css` | Core CSS and layout components |
| `@lism-css/ui` | UI components (Accordion, Modal, Tabs, etc.) |

```jsx
// React
import { Box, Flex, Stack, Grid, Text, Media } from 'lism-css/react';
import { Accordion, Tabs, Modal, Button } from '@lism-css/ui/react';

// Astro
import { Box, Flex, Stack, Grid, Text, Media } from 'lism-css/astro';

```


## CSS Layer structure

Lism CSS uses CSS Layers for specificity management.
Be aware of this order when adding custom CSS.

```
Settings (token definitions)
  â†’ @layer lism-base (Reset CSS, tokens, .set-- classes)
  â†’ @layer lism-modules (.is-- / .l-- / .a-- / .c-- module classes)
  â†’ @layer lism-utility (.u-- utility classes)
  â†’ @layer lism-custom (user customization)
  â†’ Prop Class (outside layers â€” highest specificity)
```

Details: https://lism-css.com/en/docs/css-methodology/


## Naming conventions and prefixes

The class name prefix determines its role and layer membership.

| Prefix | Layer | Role | Example |
|--------|-------|------|---------|
| `.set--` | lism-base | Override base styles, redefine tokens | `.set--plain`, `.set--transition` |
| `.is--` | lism-modules | Toggleable state modules | `.is--container`, `.is--wrapper` |
| `.l--` | lism-modules | Layout structure modules | `.l--grid`, `.l--flex`, `.l--stack` |
| `.a--` | lism-modules | Smallest layout unit modules | `.a--icon`, `.a--divider` |
| `.c--` | lism-modules | Specific-role components | `.c--button`, `.c--accordion` |
| `.u--` | lism-utility | Purpose-specific utilities | `.u--cbox`, `.u--trim` |
| `.-` | Outside layers | Single property control (Prop Class) | `.-fz:l`, `.-p:20`, `.-d:none` |

**Combination rules:**
- `.l--` and `.c--` can be combined (e.g., `<div class="l--flex c--nav">`)
- Same-category combinations are not allowed (e.g., don't use `.l--flex` and `.l--grid` on the same element)
- Variants: `.c--button.c--button--outline`
- Child elements: `.c--card_header`, `.c--card_body`

Details: https://lism-css.com/en/docs/module-class/


## How to write Props

Lism components accept CSS properties as shorthand props.
Use design tokens (described below) for values.

### Key Props

| Prop | CSS | Example |
|------|-----|---------|
| `p`, `px`, `py` | `padding` family | `p='20'`, `px='box'` |
| `m`, `mx`, `my` | `margin` family | `m='20'`, `mx='auto'` |
| `g` | `gap` | `g='20'`, `g='40'` |
| `fz` | `font-size` | `fz='l'`, `fz='2xl'` |
| `fw` | `font-weight` | `fw='bold'` |
| `lh` | `line-height` | `lh='l'` |
| `ta` | `text-align` | `ta='center'` |
| `c` | `color` | `c='main'`, `c='accent'` |
| `bgc` | `background-color` | `bgc='base-2'` |
| `d` | `display` | `d='flex'`, `d='grid'` |
| `w`, `h` | `width`, `height` | `w='100%'`, `h='auto'` |
| `bd` | `border` | `bd` (can be used alone) |
| `bdc` | `border-color` | `bdc='main'` |
| `bdrs` | `border-radius` | `bdrs='l'` |
| `bxsh` | `box-shadow` | `bxsh='2'` |
| `ar` | `aspect-ratio` | `ar='16/9'` |
| `ov` | `overflow` | `ov='hidden'` |
| `gt`, `gtc`, `gtr` | `grid-template` family | `gtc='1fr 1fr'` |
| `fx`, `fxd`, `fxw` | `flex` family | `fxd='column'` |

Full Props list: https://lism-css.com/en/docs/prop-class/


## Design tokens

Prefer design tokens over raw CSS values for prop values.

### Spacing (SPACE)
`0`, `5`, `10`, `15`, `20`, `25`, `30`, `40`, `50`
â†’ Maps to CSS variable `--s--{n}` (e.g., `--s--20`)

### Font size (FONT SIZE)
`2xs`, `xs`, `s`, `m`, `l`, `xl`, `2xl`, `3xl`, `4xl`, `5xl`
â†’ Maps to CSS variable `--fz--{key}` (e.g., `--fz--l`)

### Colors (COLOR)
- Theme: `main`, `accent`
- Background: `base`, `base-2`, `base-3`
- Text: `content`, `content-2`, `content-3`

Details: https://lism-css.com/en/docs/tokens/


## Responsive design

Lism CSS uses container queries by default.
Breakpoints: `sm: 480px`, `md: 800px`

### Using component Props (recommended)

Specify values per breakpoint using array or object syntax.

```jsx
// Array: [default, sm, md]
<Box p={[20, 30, 40]} fz={['s', 'm', 'l']} />

// Object
<Box p={{ _: '20', sm: '30', md: '40' }} />
```

### Using HTML / Prop Classes

Combine Prop Classes with CSS variables.

```html
<div class="-p:20 -p_sm -p_md" style="--p_sm: var(--s--30); --p_md: var(--s--40)">
  ...
</div>
```

Details: https://lism-css.com/en/docs/responsive/


## Adding custom CSS

When adding custom styles, write them inside `@layer lism-custom`.
This maintains consistency with Lism CSS's Layer structure.

```css
/* Add inside Lism's lism-custom layer */
@layer lism-custom {
  .my-custom-card {
    border: 1px solid var(--c--main);
    border-radius: var(--bdrs--m);
    padding: var(--s--30);
  }
}
```

Use Lism's CSS variables (tokens) in custom CSS, not raw values.

```css
/* âś• Avoid */
.my-card { padding: 24px; border-radius: 8px; }

/* â—‹ Recommended */
.my-card { padding: var(--s--30); border-radius: var(--bdrs--m); }
```

Details: https://lism-css.com/en/docs/customize/


## Implementation rules

### Use Props / classes for (let Lism handle these)
- Layout structure (layout modules like `Stack`, `Flex`, `Grid`, `Columns`)
- Token values for spacing, sizing, font sizes
- Responsive switching
- Colors, borders, shadows, and decorations

### Use native CSS for (inside @layer lism-custom)
- Animations and transitions
- Complex selectors (`:nth-child`, `::before`, `::after`, etc.)
- Custom calculations using custom properties
- Special styles not covered by Lism's tokens or modules

### Component reuse rules
- Extract as a component when the same style combination appears in 3+ places
- Build components on top of Lism's layout modules
- Follow `.c--{name}` naming convention when custom classes are needed

```jsx
// âś• Copy-pasting the same structure
<Box bgc="base-2" bdrs="l" p="30">...</Box>
<Box bgc="base-2" bdrs="l" p="30">...</Box>

// â—‹ Extract as a component
function Card({ children }) {
  return <Box bgc="base-2" bdrs="l" p="30">{children}</Box>;
}
```
