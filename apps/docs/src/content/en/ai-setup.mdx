This guide explains how to set up AI coding tools (Claude Code, Cursor, Codex, etc.) to work effectively with Lism CSS.

There are two main approaches:

| Method | Description |
|--------|-------------|
| **MCP Server** | Allows AI tools to reference component, Props, and token information in real time |
| **SKILL.md** | Place a file in your project to communicate design rules to AI agents |

Either approach works on its own, but **using both together is more effective**.


## MCP Server

`@lism-css/mcp` is an MCP (Model Context Protocol) server that provides Lism CSS documentation and API information to AI coding tools.

### What it can do

- **Component reference** — Props, usage examples, and sub-component structure for each component
- **Props to CSS mapping** — Reverse lookup of abbreviations like `p` → `padding`, `fz` → `font-size`
- **Design token reference** — Token values for colors, spacing, font sizes, and more
- **Documentation search** — Full-text search by keywords
- **CSS to Lism conversion** — Suggests Lism CSS props and components from existing CSS code

# Lism CSS Guide

This project uses the CSS framework "Lism CSS".

If the MCP server (`@lism-css/mcp`) is available, use it to get detailed component and Props information.

> **Version info:** This guide is based on `lism-css@0.11.0`. Check the version of `lism-css` used in the project, and notify the user if it differs from this guide's version.

Official documentation: https://lism-css.com/en/docs/overview/


## Package structure

| Package | Purpose |
|---------|---------|
| `lism-css` | Core CSS and layout components |
| `@lism-css/ui` | UI components (Accordion, Modal, Tabs, etc.) |

```jsx
// React
import { Box, Flex, Stack, Grid, Text, Media } from 'lism-css/react';
import { Accordion, Tabs, Modal, Button } from '@lism-css/ui/react';

// Astro
import { Box, Flex, Stack, Grid, Text, Media } from 'lism-css/astro';
import { Accordion, Tabs, Modal, Button } from '@lism-css/ui/astro';
```


## CSS Layer structure

Lism CSS uses CSS Layers for specificity management.
Be aware of this order when adding custom CSS.

```
Settings (token definitions)
  → @layer lism-base (Reset CSS, tokens, .set-- classes)
  → @layer lism-modules (.is-- / .l-- / .a-- / .c-- module classes)
  → @layer lism-utility (.u-- utility classes)
  → @layer lism-custom (user customization)
  → Prop Class (outside layers — highest specificity)
```

Details: https://lism-css.com/en/docs/css-methodology/


## Naming conventions and prefixes

The class name prefix determines its role and layer membership.

| Prefix | Layer | Role | Example |
|--------|-------|------|---------|
| `.set--` | lism-base | Override base styles, redefine tokens | `.set--plain`, `.set--transition` |
| `.is--` | lism-modules | Toggleable state modules | `.is--container`, `.is--wrapper` |
| `.l--` | lism-modules | Layout structure modules | `.l--grid`, `.l--flex`, `.l--stack` |
| `.a--` | lism-modules | Smallest layout unit modules | `.a--icon`, `.a--divider` |
| `.c--` | lism-modules | Specific-role components | `.c--button`, `.c--accordion` |
| `.u--` | lism-utility | Purpose-specific utilities | `.u--cbox`, `.u--trim` |
| `.-` | Outside layers | Single property control (Prop Class) | `.-fz:l`, `.-p:20`, `.-d:none` |

**Combination rules:**
- `.l--` and `.c--` can be combined (e.g., `<div class="l--flex c--nav">`)
- Same-category combinations are not allowed (e.g., don't use `.l--flex` and `.l--grid` on the same element)
- Variants: `.c--button.c--button--outline`
- Child elements: `.c--card_header`, `.c--card_body`

Details: https://lism-css.com/en/docs/module-class/


## How to write Props

Lism components accept CSS properties as shorthand props.
Use design tokens (described below) for values.

> The Props and classes listed here are only representative examples. Refer to the links at the end of each section for full details.

### Prop Class (HTML syntax)

JSX Props are expressed in HTML as **Prop Classes** in the format `.-{prop}:{value}`.

```jsx
// JSX
<Box p='20' fz='l' c='brand' />

// Equivalent HTML
<div class="l--box -p:20 -fz:l -c:brand">...</div>
```

For arbitrary values that don't match tokens or presets, use a class + CSS variable combination.

```html
<!-- Arbitrary value: .-{prop} class + --{prop} variable -->
<div class="-w" style="--w: 200px">...</div>
```

### Key Props

| Prop | CSS | JSX Example | HTML Prop Class |
|------|-----|-------------|-----------------|
| `p`, `px`, `py` | `padding` family | `p='20'` | `.-p:20` |
| `m`, `mx`, `my` | `margin` family | `m='20'`, `mx='auto'` | `.-m:20`, `.-mx:auto` |
| `g` | `gap` | `g='20'` | `.-g:20` |
| `fz` | `font-size` | `fz='l'` | `.-fz:l` |
| `fw` | `font-weight` | `fw='bold'` | `.-fw:bold` |
| `lh` | `line-height` | `lh='l'` | `.-lh:l` |
| `ta` | `text-align` | `ta='center'` | `.-ta:center` |
| `c` | `color` | `c='brand'` | `.-c:brand` |
| `bgc` | `background-color` | `bgc='base-2'` | `.-bgc:base-2` |
| `d` | `display` | `d='none'` | `.-d:none` |
| `w`, `h` | `width`, `height` | `w='100%'` | `.-w:100%` |
| `bd` | `border` | `bd` | `.-bd` |
| `bdc` | `border-color` | `bdc='brand'` | `.-bdc:brand` |
| `bdrs` | `border-radius` | `bdrs='20'` | `.-bdrs:20` |
| `bxsh` | `box-shadow` | `bxsh='20'` | `.-bxsh:20` |
| `ar` | `aspect-ratio` | `ar='16/9'` | `.-ar:16/9` |
| `ov` | `overflow` | `ov='hidden'` | `.-ov:hidden` |
| `pos` | `position` | `pos='sticky'` | `.-pos:sticky` |
| `ai`, `jc` | `align-items`, etc. | `ai='center'` | `.-ai:center` |
| `fxw` | `flex-wrap` | `fxw='wrap'` | `.-fxw:wrap` |
| `fxd` | `flex-direction` | `fxd='column'` | `.-fxd:col` |
| `gtc`, `gtr` | `grid-template` family | `gtc='1fr 1fr'` | `.-gtc` + `--gtc: 1fr 1fr` |

Some values are abbreviated in HTML Prop Classes (e.g., `column` → `col`, `relative` → `rel`, `absolute` → `abs`).

Full Props list: https://lism-css.com/en/docs/prop-class/


### is-- Props (State modules)

Classes that define structural behavior of elements. Belong to `@layer lism-modules`.
In HTML, apply the corresponding `is--` class directly.

| JSX Prop | HTML Class | Purpose |
|----------|-----------|---------|
| `isContainer` | `is--container` | Enable as container query container |
| `isWrapper` | `is--wrapper` | Limit content width of children (presets: `s`, `l`) |
| `isLayer` | `is--layer` | Absolutely positioned layer with `inset: 0` |
| `isLinkBox` | `is--linkBox` | Make entire box clickable as a link |
| `isVertical` | `is--vertical` | Vertical writing mode |

```jsx
// JSX
<Box isContainer isWrapper='l'>...</Box>

// Equivalent HTML
<div class="l--box is--container is--wrapper -contentSize:l">...</div>
```

Details: https://lism-css.com/en/docs/state/


### set-- Props (Base setup)

Classes that set up CSS variables and base styles to enable features. Belong to `@layer lism-base`.
In HTML, apply the corresponding `set--` class directly.

| JSX Prop | HTML Class | Purpose |
|----------|-----------|---------|
| `setPlain` | `set--plain` | Reset default styles for buttons, links, etc. |
| `setShadow` | `set--shadow` | Redefine shadow token variables (use with `bxsh`) |
| `setHov` | `set--hov` | Set hover state detection variables |
| `setTransition` | `set--transition` | Set transition CSS variables |
| `setGutter` | `set--gutter` | Apply horizontal padding (`--gutter-size`) |
| `setBp` | `set--bp` | Set breakpoint detection variables |

```jsx
// JSX
<Box setShadow bxsh='20'>...</Box>

// Equivalent HTML
<div class="l--box set--shadow -bxsh:20">...</div>
```

Details: https://lism-css.com/en/docs/set/


## Design tokens

Prefer design tokens over raw CSS values for prop values.

### Spacing (SPACE)
`5`, `10`, `15`, `20`, `30`, `40`, `50`, `60`, `70`, `80`
→ Maps to CSS variable `--s{n}` (e.g., `--s20`)

### Font size (FONT SIZE)
`2xs`, `xs`, `s`, `m`, `l`, `xl`, `2xl`, `3xl`, `4xl`, `5xl`
→ Maps to CSS variable `--fz--{key}` (e.g., `--fz--l`)

### Border radius (BORDER RADIUS)
`10`, `20`, `30`, `40`, `99`
→ Maps to CSS variable `--bdrs--{n}` (e.g., `--bdrs--20`)

### Box shadow (BOX SHADOW)
`10`, `20`, `30`, `40`
→ Maps to CSS variable `--bxsh--{n}` (e.g., `--bxsh--20`)

### Semantic colors (COLOR)
- Theme: `brand`, `accent`
- Background: `base`, `base-2`
- Text: `text`, `text-2`
- Other: `divider`, `link`
→ Maps to CSS variable `--{name}` (e.g., `--brand`, `--text`)

### Palette colors (PALETTE)
`red`, `blue`, `green`, `yellow`, `purple`, `orange`, `pink`, `gray`, `white`, `black`, `keycolor`
→ Maps to CSS variable `--{name}` (e.g., `--red`, `--blue`)
→ When specified via Props, internally processed as `var(--palette--{name})`

Details: https://lism-css.com/en/docs/tokens/


## Responsive design

Lism CSS uses container queries by default.
Breakpoints: `sm: 480px`, `md: 800px`

### Using component Props (recommended)

Specify values per breakpoint using array or object syntax.

```jsx
// Array: [default, sm, md]
<Box p={[20, 30, 40]} fz={['s', 'm', 'l']} />

// Object
<Box p={{ base: '20', sm: '30', md: '40' }} />
```

### Using HTML / Prop Classes

Combine Prop Classes with CSS variables.

```html
<div class="-p:20 -p_sm -p_md" style="--p_sm: var(--s30); --p_md: var(--s40)">
  ...
</div>
```

Details: https://lism-css.com/en/docs/responsive/


## Adding custom CSS

When adding custom styles, write them inside the appropriate Lism CSS Layer.

```css
/* Custom components → add to lism-modules */
@layer lism-modules {
  .c--my-card {
    border: 1px solid var(--brand);
    border-radius: var(--bdrs--20);
    padding: var(--s30);
  }
}

/* Base style extensions → add to lism-base */
@layer lism-base {
  .set--my-theme {
    --brand: #c00;
  }
}
```

Use Lism's CSS variables (tokens) in custom CSS, not raw values.

```css
/* ✕ Avoid */
.c--my-card { padding: 24px; border-radius: 8px; }

/* ○ Recommended */
.c--my-card { padding: var(--s30); border-radius: var(--bdrs--20); }
```

Details: https://lism-css.com/en/docs/customize/


## Implementation rules

### Use Props / classes for (let Lism handle these)
- Layout structure (layout modules like `Stack`, `Flex`, `Grid`, `Columns`)
- Token values for spacing, sizing, font sizes
- Responsive switching
- Colors, borders, shadows, and decorations

### Use native CSS for (inside appropriate @layer)
- Animations and transitions
- Complex selectors (`:nth-child`, `::before`, `::after`, etc.)
- Custom calculations using custom properties
- Special styles not covered by Lism's tokens or modules

### Component reuse rules
- Extract as a component when the same style combination appears in 3+ places
- Build components on top of Lism's layout modules
- Follow `.c--{name}` naming convention when custom classes are needed

```jsx
// ✕ Copy-pasting the same structure
<Box bgc="base-2" bdrs="20" p="30">...</Box>
<Box bgc="base-2" bdrs="20" p="30">...</Box>

// ○ Extract as a component
function Card({ children }) {
  return <Box bgc="base-2" bdrs="20" p="30">{children}</Box>;
}
```
