This guide explains how to set up and use Lism CSS.


There are two main approaches to getting started.

## CSS-only setup

For simple HTML sites or other cases where you only need the CSS file, you can use it via CDN.


<PreviewTitle>Load via CDN</PreviewTitle>
```html
<link href="https://cdn.jsdelivr.net/npm/lism-css@0.9.4/dist/css/main.css" rel="stylesheet" />
```

You can also download the CSS file and load it locally.


## Using dedicated components

React (.jsx) and Astro (.astro) components are distributed as an npm package.

### 1. Install the package

<Preview>
	<PreviewTitle>Install the package</PreviewTitle>
	<PreviewCode slot='tab' label='npm'>
	```bash frame="none"
	npm i lism-css
	```
	</PreviewCode>
	<PreviewCode slot='tab' label='pnpm'>
	```bash frame="none"
	pnpm add lism-css
	```
	</PreviewCode>
	<PreviewCode slot='tab' label='yarn'>
	```bash frame="none"
	yarn add lism-css
	```
	</PreviewCode>
</Preview>




### 2. Import the CSS

<PreviewTitle>Import it as a global style.</PreviewTitle>
```js
import 'lism-css/main.css';
```

For example, in Next.js, import it in `_app.js` or `layout.js`.



### 3. Import components

<Preview>
<PreviewTitle>Import individual components</PreviewTitle>
<PreviewCode slot='tab' label='.jsx'>
```jsx
import { Box, Flex, ... } from 'lism-css/react';
```
</PreviewCode>
<PreviewCode slot='tab' label='.astro'>
```jsx
import { Box, Flex, ... } from 'lism-css/astro';
```
</PreviewCode>
</Preview>



:::warning
::title[Note when importing .astro files]

Directly importing `.astro` files from inside `node_modules` may not work and could require additional configuration.
If you encounter a "**Directory import '...' is not supported resolving ES modules** ..." error, add `"lism-css"` to `vite.ssr.noExternal` in your `astro.config.js`.

<PreviewTitle>`astro.config.js`</PreviewTitle>
```js
import { defineConfig } from 'astro/config';

export default defineConfig({
  // ...other settings
  vite: {
    ssr: {
      noExternal: ['lism-css'],
    },
  },
});
```


<small>(We are investigating whether this can be resolved by improving the package distribution method...)</small>
:::

## AI Tool Integration (MCP)

An MCP server is available that allows AI coding tools to reference the Lism CSS documentation and API.

<Preview>
	<PreviewCode slot='tab' label='Claude Code'>
	```bash frame="none"
	claude mcp add lism-css -- npx -y @lism-css/mcp
	```
	</PreviewCode>
	<PreviewCode slot='tab' label='Cursor'>
	```json frame="none"
	// .cursor/mcp.json
	{
	  "mcpServers": {
	    "lism-css": {
	      "command": "npx",
	      "args": ["-y", "@lism-css/mcp"]
	    }
	  }
	}
	```
	</PreviewCode>
	<PreviewCode slot='tab' label='VS Code'>
	```json frame="none"
	// .vscode/mcp.json
	{
	  "servers": {
	    "lism-css": {
	      "type": "stdio",
	      "command": "npx",
	      "args": ["-y", "@lism-css/mcp"]
	    }
	  }
	}
	```
	</PreviewCode>
</Preview>

For details, see [GitHub - @lism-css/mcp](https://github.com/lism-css/lism-css/tree/main/packages/mcp).
