---
title: 'Module Class'
description: 'An overview of Lism CSS module classes (is--, l--, a--, c--) and their usage.'
---


Lism uses the following module classes.

- `is--{module}`: Versatile state modules that can be applied or removed across multiple types of elements.
- `l--{module}`: Modules that serve as the building blocks of layouts.
- `a--{module}`: Modules that represent the smallest units of a layout.
- `c--{module}`: Components with more specific roles. (Many are designed to be used in combination with other classes.)


In general, multiple module classes from different layers cannot be combined, except for State Modules.
For example, `.l--` and `.c--` can be used together, but `.l--XXX` and `.l--YYY`, or `.c--XXX` and `.c--YYY`, should not be combined within the same element.


## State Modules

Versatile state modules that can be applied or removed across multiple types of elements.

| Class | Description |
|---|---|
| <ModLink slug="is--container" /> | Defines the reference element for container queries |
| <ModLink slug="is--wrapper" /> | Wrapper element that controls content width |
| <ModLink slug="is--layer" /> | Overlay layer positioned over a parent element |
| <ModLink slug="is--boxLink" /> | Makes the entire box a clickable link |
| <ModLink slug="is--vertical" /> | Applies vertical writing mode |
| `is--side` | For the side element of `l--sideMain` (→ see <ModLink slug="l--sidemain" />) |
| `is--skipFlow` | Cancels spacing with the next sibling inside `l--flow` (→ see <ModLink slug="l--flow" />) |


## Layout Modules

Modules that serve as the building blocks of layouts.

| Class | Description |
|---|---|
| <ModLink slug="l--box" /> | Simple module for grouping content |
| <ModLink slug="l--flow" /> | Flow-root module that auto-manages spacing between children |
| <ModLink slug="l--frame" /> | Fits child media to its own size |
| <ModLink slug="l--flex" /> | Lays out content using Flexbox |
| <ModLink slug="l--cluster" /> | Arranges items horizontally with automatic wrapping |
| <ModLink slug="l--stack" /> | Arranges elements vertically |
| <ModLink slug="l--sidemain" /> | Two-column layout with side and main areas |
| <ModLink slug="l--grid" /> | Lays out content using CSS Grid |
| <ModLink slug="l--tilegrid" /> | Uniform row-column grid (tile layout) |
| <ModLink slug="l--center" /> | Centers content both horizontally and vertically |
| <ModLink slug="l--columns" /> | Column layout with breakpoint-based column counts |
| <ModLink slug="l--fluidcols" /> | Fluid columns that wrap based on column width |
| <ModLink slug="l--switchcols" /> | Automatically switches between multi-column and single-column |


## Atomic Modules

Modules that represent the smallest units of a layout.

| Class | Description |
|---|---|
| <ModLink slug="a--decorator" /> | Empty element for decoration |
| <ModLink slug="a--divider" /> | Divider line between content |
| <ModLink slug="a--icon" /> | Outputs an SVG icon |
| <ModLink slug="a--spacer" /> | Empty element for spacing between elements |
