import { Container, Wrapper, Box, Lism, Spacer } from "lism-css/astro";

`container-type`をセットできる[`is--container`](/docs/css-methodology//#is--container) クラスをコンポーネントから利用する例を紹介していきます。

`is--container` は基本的に `isContainer`プロパティで**どのコンポーネントからでも呼び出せるようになっています。**

| プロパティ | 出力 |
|---|---|
| `isContainer`| `.is--container` |

`<Lism isContainer>` のエイリアスとして、`<Container>` も用意しています。

ここでは、この専用コンポーネントを紹介していきます。


## Props

`<Container>` で使用できるプロパティは次のとおりです。


| プロパティ | 説明 |
|---|---|
| `size` | `isWrapper` に渡すコンテンツサイズを指定します。プリセット値 (`'s'`, `'l'`) や任意の CSS 値を指定できます。|
| `layout` | 展開したい Layoutモジュールを指定できます。|

<PreviewTitle>下記の ①, ② はそれぞれ同じ結果になります。</PreviewTitle>

```jsx
// ①
<Flow isContainer>...</Flow>

// ②
<Container layout='flow'>...</Container>
```


## Import

<ImportPackage component='Container' />


## Usage

`isContainer`をLismコンポーネントにセットするか、または`<Container>`コンポーネントを使用して`is--container`が付与され要素の内部では、コンテナクエリが有効になります。
