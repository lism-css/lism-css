Lism CSSでは、`line-height` の扱いが独特となっており、**ハーフレディングの値でline-heightを管理する**仕組みになっています。

ハーフレディングとは、`line-height`から文字サイズを除いた上下の余白部分のことです。


## 仕組み

Lism では、全要素に対し、以下のスタイルが適用されています。

```css
* {
  line-height: calc(1em + var(--hl) * 2);
}
```

これにより、`line-height`の値は、`--hl` というハーフレディングを指定するための変数に依存する仕組みになっており、
**どんな文字サイズのテキストに対しても同じ余白量**でレイアウトを組んでいくことが可能です。


## トークン

`--hl--` でトークンを用意しています。

|トークン|変数|初期値
|---|---|---|
|`xs`|`--hl--xs`|`var(--hl-unit)`|
|`s`|`--hl--s`|`calc(var(--hl-unit) * 2)`|
|`base`|`--hl--base`|`calc(var(--hl-unit) * 3)`|
|`l`|`--hl--l`|`calc(var(--hl-unit) * 4)`|

上記の`--hl-unit`は次のように定義されています。

```css
:root {
  --hl-unit: calc(var(--fz--base) * 0.125); /* ≒ 2px */
}
```

（`--fz--base` はデフォルト `1rem` のため、`--hl-unit` は `2px`前後の値となる想定です。）


## `hl` クラス

ハーフレディングは `hl`（half-leading）プロパティで指定します。`-hl:*` クラス（`<Lism>` 系コンポーネントでは `hl` prop）の内部で、これらの変数を `--hl` にセットしています。

```css
.-hl\:base {
  --hl: var(--hl--base);
}
.-hl\:xs {
  --hl: var(--hl--xs);
}
.-hl\:s {
  --hl: var(--hl--s);
}
.-hl\:l {
  --hl: var(--hl--l);
}
.-hl\:0 {
  --hl: 0px; /* ハーフレディングなし */
}
```

`hl` はブレイクポイント指定にも対応しており、`hl={['s', null, 'l']}`（`-hl_md:l`）のようにレスポンシブに切り替えられます。


### `lh`（互換ショートカット）

`line-height` の感覚で指定したい場合のために、`lh` を互換ショートカットとして用意しています。

|指定|挙動|
|---|---|
|`lh="base"` / `"xs"` / `"s"` / `"l"`|`hl` と同じトークンを参照（`--hl` を制御）|
|`lh="1"`|ハーフレディングなし（`hl="0"` と等価）|
|`lh="1.7"` などの任意の数値|従来どおり CSS の `line-height` をそのまま出力|

公式の例では基本的に `hl` を使用します。`line-height` の数値をそのまま指定したいときだけ `lh` を利用してください。


## ハーフレディング管理のメリット

line-heightを`1.6`などの一般的な値でセットした場合、フォントサイズが大きくなるにつれて、line-heightの値も大きくなってしまいます。

ベースサイズ付近のテキストであればそれでいいのですが、見出しなどになるとこれでは大きすぎて読みづらいという問題が発生してきます。

行間のハーフレディングを、固定値で管理することで、フォントサイズが大きくなってもテキスト間の距離が一定に保たれます。
