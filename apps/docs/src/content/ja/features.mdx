## サイト全体のCSS設計を担う

Lism CSS は単なるユーティリティクラスのパッケージではありません。

- デザイントークンの設計
- クラス名と変数名の命名規則
- `reset.css` レベルからの**レイヤー(`@layer`)階層**

これらを包括的に定義しており、**サイト全体のCSS設計を丸ごと引き受ける**フレームワークです。

CSS Layers による詳細度管理も行われており、以下の階層構造で整理されています。

```
lism-reset → lism-base → lism-modules → lism-utility
```

Lism の設計を一度学べば、以後は自分でCSS設計を考える必要がなくなります。  
そのかわり、他のCSSフレームワークとの共存は想定していません。


## 心地よいリズムを生むトークン設計

余白やフォントサイズのスケーリングにこだわったデザイントークンを用意しています。

トークンを使い回すことで、意識せずとも**デザインに自然なリズム**が生まれます。  
メンテナンス性と一貫性の向上にもつながる設計です。


## レイアウトファースト

WEBサイトで一般的によく使われるレイアウト要素を細分化し、汎用的に使い回せる設計になっています。

これは、[Every Layout](https://every-layout.dev/) での**レイアウトプリミティブ**という思想を参考にしたものです。

- `l--flex` — Flexboxベースのレイアウト
- `l--stack` — 縦積みレイアウト
- `l--columns` — カラムレイアウト
- `l--center` — 中央寄せレイアウト
- `l--switchCols` — コンテナ幅に応じて自動で段組みが切り替わるレイアウト

など、多くのレイアウトモジュールが定義されています。


## CSS変数を活かしたユーティリティ設計

Lism には Tailwind CSS のように、個々のCSSプロパティに対応するユーティリティクラスがあります。  
ただし、すべてのプロパティ × すべてのブレークポイントのクラスを用意するわけではありません。

**CSS変数（カスタムプロパティ）と組み合わせる独自の方式**により、少ないCSS容量のまま柔軟にレスポンシブ対応ができます。

たとえば、`-p:20` のような基本的なトークンクラスは用意されていますが、`md:p-30` のようなブレークポイント付きクラスは提供しません。  
かわりに、`-p_md` クラスと `--p_md` 変数を組み合わせることで対応します。

:::check
このようにして、主要なCSSプロパティに関して、**簡単にブレイクポイントでの切り替えができる**ようになっています。
:::


## CSSを書くことを前提にした設計思想

Lism CSS は、HTMLに全部書く Tailwind CSS とも、JSXにスタイルを閉じ込める CSS-in-JS とも異なるアプローチを取っています。

- **主要なレイアウトや軽い装飾** → クラスとCSS変数でサクっと組む
- **細部のスタイリングや作り込み** → ネイティブCSSで丁寧に整える

このバランスを大切にした設計です。  
すべてをクラスで完結させるのではなく、CSSを書くべきところはしっかり書くという前提になっています。


## メディアクエリに依存しないレイアウト

Lism CSS では、メディアクエリではなく**コンテナクエリをデフォルトで採用**しています。

さらに、[Every Layout](https://every-layout.dev/) の思想を取り入れ、**クエリに依存せず、コンテナの幅に応じて自動的にレイアウトが切り替わるモジュール**もいくつか用意しています。

- `l--fluidCols` — コンテナ幅に応じてカラム数が自動調整
- `l--switchCols` — コンテナ幅に応じて段組みとスタックが自動切替

もちろん、ブレークポイントに基づく従来のレスポンシブ設計も併用できます。

:::warning
コンテナ要素の中では `position: fixed` の挙動が変わったり、サイズ関係の挙動が少し変わったりすることがあるため注意が必要です。
:::


## ゼロビルドでどんな環境でも使える

Lism CSS はあくまでも**HTMLとCSSの設計理論が主体**になっており、特殊なビルド処理は必須ではありません。

:::check
CSSを読み込むだけでも導入でき、**素のHTMLサイトでも利用できる**ものになっています。
:::

CDNから1行で読み込むだけでも使い始められます。

```html
<link href="https://cdn.jsdelivr.net/npm/lism-css@0/dist/css/main.css" rel="stylesheet" />
```

ただし、React や Astro 向けの**専用コンポーネント**を使うことで、さらに快適な開発体験が得られます。


## UIコンポーネントは別パッケージ

Lism CSS のコアパッケージ（`lism-css`）には、具体的なスタイリングが施されたUI系コンポーネント（アコーディオン、タブ、モーダルなど）は含まれていません。

それらは `@lism-css/ui` という**別パッケージ**として、あくまで雛形の位置づけで提供しています。

- パッケージをそのまま `import` して使う
- CLIコマンドで自分のプロジェクトにコピーし、自由にカスタマイズして使う

どちらの使い方にも対応しています。

## Inspirations

Lism CSS は以下のプロジェクトから多くのインスピレーションを受けています。

- [Every Layout](https://every-layout.dev/) — "**レイアウトプリミティブ**"という考え方をベースに取り入れています。
- [ITCSS](https://itcss.io/) — レイヤーの概念を参考にしています。
- [Tailwind CSS](https://tailwindcss.com/) — CSSプロパティと値のセットで細かくクラス化するという考え方を参考にしています。
- [daisyUI](https://daisyui.com/) — カラートークンの設計やコンポーネントの豊富さが参考になりました。
- [SmartHR Design System](https://smarthr.design/products/design-tokens/) — デザイントークン設計が参考になりました。
- [Chakra UI](https://chakra-ui.com/), [MUI](https://mui.com/) — コンポーネントの使用感が参考になりました。
