アコーディオン要素を作成できるUIコンポーネントです。

`<details>` / `<summary>` ではなく、`<div>` / `<button>` ベースで構成されており、JavaScriptで開閉アニメーションを制御します。
パネルには `hidden="until-found"` を活用しているため、アコーディオンが閉じた状態でもブラウザのページ内検索でコンテンツを検出でき、ヒット時に自動的にパネルが展開されます。

## Import

`@lism-css/ui` パッケージで提供しています。

<ImportPackage component='Accordion' script='setAccordion.js' packageName='@lism-css/ui' />

以下のコンポーネントが利用できます。

- `<Accordion.Root>`: アコーディオンのルート要素
- `<Accordion.Heading>`: 見出しエリアのラッパー（デフォルトは `<div role="heading">`）
- `<Accordion.Button>`: 開閉トリガーのボタン（末尾に `<Accordion.Icon>` を自動で含みます）
- `<Accordion.Icon>`: 開閉状態を示すアイコン（CSS疑似要素で描画）
- `<Accordion.Panel>`: 開閉されるコンテンツパネル（`hidden="until-found"` を使用）


## Props

| プロパティ | 説明 |
|---|---|
| <PropBadge>`<Accordion.Heading>`</PropBadge><br/>`tag` | 見出しのHTMLタグを指定します。デフォルトは `div`（`role="heading"` が自動付与されます）。`h2`〜`h6` を指定すると `role` は付与されません。 |
| <PropBadge>`<Accordion.Panel>`</PropBadge><br/>`flow` | パネル内のコンテンツ領域（`__content`）に渡すフローレイアウトの設定。 |

### アニメーション時間を変更し、複数同時展開を許可する

アコーディオンの開閉アニメーションの速度は `--duration` 変数で管理されており、デフォルトは `0.25s` です。

:::note
デフォルトでは、同じ親要素内で一つのアコーディオンしか同時に展開できません。  
親要素に `data-allow-multiple` 属性を指定することで、複数を同時に展開できるようになります。
:::

{/* ## スクリプトの処理の内容について

おおまかな処理ステップは以下の通りです。

- `.c--accordion` 内の `button` 要素に対して `click` イベントを登録
- `.c--accordion_panel` に対して `beforematch` イベントを登録（ブラウザのページ内検索時に自動展開）
- クリック時、開閉状態に応じて `data-opened` 属性と `aria-expanded` 属性を操作
    - `--height--opened` 変数で高さを制御し、アニメーションの終了を待ってから状態を確定
	- アコーディオンが開く時、親要素に `data-allow-multiple` がなければ、他の兄弟アコーディオンを閉じる
- アニメーション完了後、閉じた場合は `hidden="until-found"` を再付与 */}
