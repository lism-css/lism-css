アコーディオン要素を作成できるUIコンポーネントです。

`<details>` / `<summary>` ではなく、`<div>` / `<button>` ベースで構成されており、JavaScriptで開閉アニメーションを制御します。
パネルには `hidden="until-found"` を活用しているため、アコーディオンが閉じた状態でもブラウザのページ内検索でコンテンツを検出でき、ヒット時に自動的にパネルが展開されます。

## How to use

`@lism-css/ui` パッケージで`Accordion`として提供しています。


### Import

<ImportPackage component="Accordion" script="accordion.js" packageName="@lism-css/ui" />

以下のコンポーネントが利用できます。

- `<Accordion.Root>`: 複数のアコーディオンアイテムをラップするルート要素（`Stack` ベース）
- `<Accordion.Item>`: 個別のアコーディオン要素
- `<Accordion.Heading>`: 見出しエリアのラッパー（デフォルトは `<div role="heading">`）
- `<Accordion.Button>`: 開閉トリガーのボタン（末尾に `<Accordion.Icon>` を自動で含みます）
- `<Accordion.Icon>`: 開閉状態を示すアイコン（CSS疑似要素で描画）
- `<Accordion.Panel>`: 開閉されるコンテンツパネル（`hidden="until-found"` を使用）


### Props

| プロパティ | 説明 |
|---|---|
| <PropBadge>`<Accordion.Root>`</PropBadge><br/>`allowMultiple` | 複数のアコーディオンを同時に展開できるようにします。デフォルトでは一つしか展開できません。 |
| <PropBadge>`<Accordion.Item>`</PropBadge><br/>`isOpen` | アイテムを初期状態で展開します（`data-opened` を付与）。`<Accordion.Button>` / `<Accordion.Panel>` の `isOpen` と併せて指定します。 |
| <PropBadge>`<Accordion.Heading>`</PropBadge><br/>`as` | 見出しのHTMLタグを指定します。デフォルトは `div`（`role="heading"` が自動付与されます）。`h2`〜`h6` を指定すると `role` は付与されません。 |
| <PropBadge>`<Accordion.Button>`</PropBadge><br/>`isOpen` | 初期展開状態に合わせて `aria-expanded` を設定します。 |
| <PropBadge>`<Accordion.Panel>`</PropBadge><br/>`flow` | パネル内のコンテンツ領域（`__content`）に渡すフローレイアウトの設定。 |
| <PropBadge>`<Accordion.Panel>`</PropBadge><br/>`isOpen` | パネルを初期表示します（`hidden` を解除）。 |


### 基本構造

```jsx
<Accordion.Root>
  <Accordion.Item>
    <Accordion.Heading>
      <Accordion.Button>Accordion Label</Accordion.Button>
    </Accordion.Heading>
    <Accordion.Panel p="20" pt="5">
      ...Contents...
    </Accordion.Panel>
  </Accordion.Item>
</Accordion.Root>
```

:::note
`<Accordion.Button>` には、末尾に `<Accordion.Icon>` が自動的に配置されます。
アイコンはCSSの疑似要素（`::before` / `::after`）で描画されるため、SVGの指定は不要です。
:::

### アニメーション時間を変更する

`<Accordion.Root>`（全体）または `<Accordion.Item>`（個別）に CSS変数として指定することで変更できます。

### 最初の一つを開いた状態にする

初期状態で展開しておきたいアイテムには、`<Accordion.Item>` / `<Accordion.Button>` / `<Accordion.Panel>` のそれぞれに `isOpen` を指定します。

:::note
`isOpen` はコンポーネントごとに役割が異なるため、3つ揃えて指定します。
`<Accordion.Item>` では `data-opened` を付与してパネルの高さとアイコンを開いた状態にし、`<Accordion.Button>` では `aria-expanded` を、`<Accordion.Panel>` では `hidden` の切り替えを担当します。
:::

### 複数同時展開を許可する

:::note
デフォルトでは、同じ `<Accordion.Root>` 内で一つのアコーディオンしか同時に展開できません。
`allowMultiple` を指定することで、複数を同時に展開できるようになります。
:::
