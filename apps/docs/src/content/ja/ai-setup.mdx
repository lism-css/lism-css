AIコーディングツール（Claude Code, Cursor, Codex など）で Lism CSS を効率よく扱うためのセットアップ方法を解説します。

大きく分けて、以下の2つの方法があります。

| 方法 | 特徴 |
|------|------|
| **MCP Server** | AIツールがコンポーネント・Props・トークンの情報をリアルタイムに参照できる |
| **SKILL.md** | プロジェクトにファイルを配置し、AIに設計ルールを伝える |

どちらか片方だけでも機能しますが、**併用するとより効果的**です。


## MCP Server

`@lism-css/mcp` は、Lism CSS のドキュメント・API情報をAIコーディングツールに提供する MCP (Model Context Protocol) サーバーです。

### できること

- **コンポーネント情報の参照** — 各コンポーネントのProps・使用例・サブコンポーネント構成
- **Props ↔ CSS マッピング** — `p` → `padding`、`fz` → `font-size` など、省略記法の逆引き
- **デザイントークンの参照** — 色、余白、フォントサイズなどのトークン値
- **ドキュメント検索** — キーワードによる全文検索
- **CSS → Lism 変換** — 既存CSSコードからLism CSSのprops・コンポーネント提案

### セットアップ

## SKILL.md

プロジェクトに SKILL.md ファイルを配置することで、AIエージェントに Lism CSS の設計ルール・Props・トークン・命名規則などを伝えることができます。

### ファイル構成

スキルは以下の4ファイルで構成されています。

| ファイル | 内容 |
|---------|------|
| `SKILL.md` | エントリポイント — 概要・パッケージ構成・実装ルール・参照リスト |
| `props.md` | Props システム — 主要Props一覧・Prop Class記法・is--/set-- Props・レスポンシブ対応 |
| `tokens.md` | デザイントークン — 余白・フォントサイズ・角丸・影・カラー・パレット |
| `css-rules.md` | CSS設計ルール — Layer構造・命名規則・プレフィックス・カスタムCSS追加ルール |

GitHub: [lism-css/lism-css/.claude/skills/lism-css-best-practices/](https://github.com/lism-css/lism-css/tree/main/.claude/skills/lism-css-best-practices)
